# /init-docs - 初始化项目文档

作为项目文档初始化助手，请帮我创建标准化的项目管理文档。

## 你需要做的：

1. **创建 docs 目录**
   - 在项目根目录创建 `docs/` 文件夹

2. **创建需求文档**
   - 创建 `docs/需求文档.md`
   - 包含项目概述、功能需求、非功能需求、开发计划

3. **创建项目状态文档**
   - 创建 `docs/项目状态.md`
   - 包含当前状态、已完成、进行中、待办、里程碑

4. **创建待办清单**
   - 创建 `docs/待办清单.md`
   - 按优先级组织任务（高/中/低）

5. **创建 CLAUDE.md 配置**
   - 创建项目根目录的 `CLAUDE.md`
   - 包含项目资源概览、开发规范、项目管理命令

## 输出格式：

```
# 📝 项目文档初始化完成

## 已创建的文档

✅ docs/需求文档.md
✅ docs/项目状态.md
✅ docs/待办清单.md
✅ CLAUDE.md

## 文档说明

### 需求文档
- 用途：明确项目要做什么
- 包含：功能需求、非功能需求、开发计划
- 更新频率：需求变更时更新

### 项目状态
- 用途：记录项目做到哪了
- 包含：已完成、进行中、待办任务
- 更新频率：使用 /update-status 自动更新

### 待办清单
- 用途：规划接下来做什么
- 包含：按优先级组织的任务列表
- 更新频率：每天或每周更新

### CLAUDE.md
- 用途：AI 项目管理配置
- 包含：项目结构、开发规范、管理命令
- 更新频率：项目结构变化时更新

## 🎯 下一步

现在你可以使用以下命令：

1. `/start` - 快速了解项目状态
2. `/progress` - 查看详细项目进度
3. `/next` - 获取下一步开发建议
4. `/update-status` - 自动更新项目状态

## 💡 建议工作流

每天开始：
1. /start - 进入工作状态
2. /next - 获取今日任务

开发完成：
1. git commit - 提交代码
2. /update-status - 更新项目状态

需要汇报：
1. /progress - 生成进度报告
```

## 注意事项：

- 根据项目实际情况调整文档内容
- 确保文档结构清晰、易于维护
- 文档使用中文编写
- 创建完成后给出使用指导
