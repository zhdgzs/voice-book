# /start - 快速了解项目

作为项目引导助手，请帮我快速了解 Voice Book 项目的当前状态。

## 你需要做的：

1. **识别项目类型**
   - 项目名称：Voice Book
   - 技术栈：Flutter + Provider + SQLite + Just Audio
   - 项目类型：离线本地有声书播放器

2. **查看最近 Git 提交**
   - 查看最近 3 条 Git 提交记录
   - 了解最近的开发动态

3. **检查代码状态**
   - 检查是否有未提交的修改
   - 查看当前分支状态

4. **扫描项目模块**
   - 扫描 lib/ 目录下的模块结构
   - 统计已实现的功能模块

5. **统计待办事项**
   - 扫描代码中的 TODO 和 FIXME 注释
   - 统计数量和位置

## 输出格式：

请以简洁友好的方式输出以下内容：

```
# 👋 欢迎回到 Voice Book 项目

## 📂 项目信息
- **项目名称**：Voice Book
- **技术栈**：Flutter + Provider + SQLite + Just Audio
- **主要功能**：离线本地有声书播放器
- **核心理念**：低资源占用 + 隐私保护

## 🕐 最近动态
[最近 3 条 Git 提交记录]

## 📊 当前状态
- **代码状态**：[✅ 无未提交修改 / ⚠️ 有未提交修改]
- **当前分支**：[分支名称]
- **项目模块**：[已实现的模块数量]
- **待办事项**：[TODO/FIXME 数量]

## 🎯 你可以：
1. 输入 `/progress` - 查看详细的项目进度
2. 输入 `/next` - 获取下一步开发建议
3. 直接告诉我："继续开发 [功能名称]"
```

## 注意事项：

- 输出要简洁明了，不要过于冗长
- 重点突出最近的开发动态和当前状态
- 如果是新打开的项目，要特别提示上次做到哪里了
- 语气要友好、专业
